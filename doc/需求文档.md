## 1. 文档信息
- 文档名称：4G设备监控检测与去控模拟系统需求规格说明书（SRS）
- 版本：v1.1
- 类型：Web前端单体应用（模拟业务）
- 目标读者：产品经理、前端开发、测试人员、演示使用人员

## 2. 项目背景与目标
本系统用于模拟“4G设备被远程监控后的检测与去控处理流程”，提供接近运维控制台的视觉与交互体验。  
系统目标如下：
- 展示完整的“检测 -> 去控 -> 结果反馈”链路。
- 提供专业化控制台日志输出，体现通信与安全术语。
- 提供独立的监控视频展示区，用于演示场景，不与去控状态耦合。

## 3. 范围定义
### 3.1 In Scope（本期范围）
- 设备监控状态自动检测（模拟接口）。
- 一键去控流程执行（模拟多步骤接口）。
- 控制台日志弹窗（可拖动、居中、可关闭、自动滚动、进度展示）。
- 监控视频展示区（真实视频播放、全屏）。
- 基础响应式布局（PC优先，兼容常见笔记本分辨率）。

### 3.2 Out of Scope（非本期范围）
- 真实设备接入与真实通信协议对接。
- 用户认证、权限管理、审计留痕后端服务。
- 多租户、多设备并发批量处理。

## 4. 用户角色与使用场景
- 演示人员：用于方案演示，关注页面视觉效果与操作流畅性。
- 运维人员（模拟角色）：关注日志专业性、步骤可追踪性、状态可读性。
- 测试人员：关注流程完整性、交互一致性、异常提示可验证性。

典型场景：
1. 用户进入系统，自动完成检测并显示当前状态。  
2. 用户点击“一键去控”，系统弹出控制台并输出分步骤日志与进度。  
3. 用户观察日志和结果提示；监控视频持续展示，不受去控状态影响。  

## 5. 功能需求
### 5.1 设备检测模块
- 页面初始化后自动触发检测请求。
- 检测接口返回`monitored=true`时，界面显示“设备处于远程监控状态”。
- 检测完成后可触发“一键去控”。
- 支持“重新检测”动作。

### 5.2 一键去控模块
- 点击按钮后打开控制台弹窗，并开始流程执行。
- 去控流程包含多个步骤，逐步调用模拟接口。
- 每个步骤输出以下信息：
  - 步骤序号/名称
  - 执行状态（进行中、成功）
  - 时间戳
  - 专业术语与关键英文指标（示例：RSRP、SINR、PDP、RTP、DPI、NOC_ACK、TRACE_ID）
- 控制台展示整体进度条（0-100%）。
- 全流程结束后提示“去控完成，设备已解除监控”。

### 5.3 控制台弹窗模块
- 弹窗默认居中显示。
- 支持拖动移动，支持关闭。
- 日志区域自动滚动到最新行。
- 弹窗再次打开时应重新居中。

### 5.4 监控视频展示模块
- 页面提供独立监控视频区域，作为展示用途。
- 默认播放本地视频文件：`public/videos/cctv.mp4`。
- 视频区域支持“全屏”切换。
- 监控视频与去控流程状态解耦，不因去控成功/失败切换画面。

## 6. 页面与交互要求
- 页面主体居中展示，采用左侧操作区 + 右侧监控区布局。
- 监控区宽度可配置，默认采用较窄展示以突出操作面板。
- 关键操作按钮状态清晰（可用、禁用、加载中）。
- 反馈原则：每个关键动作均有可感知反馈（文案、日志、进度、消息提示）。

## 7. 接口与数据约定（模拟）
### 7.1 检测接口
- `GET /api/detect`
- 响应示例：
```json
{ "monitored": true }
```

### 7.2 去控步骤接口
- `POST /api/unlock/{stepNo}`
- 响应示例：
```json
{ "success": true }
```

### 7.3 时延策略
- 每步请求模拟延迟：200-500ms 随机。
- 日志输出应体现“实时处理感”，避免瞬时全部刷出。

## 8. 非功能需求
### 8.1 可用性
- 页面首屏加载后可在3秒内进入可操作状态（本地环境）。
- 操作路径简洁，核心操作不超过2次点击。

### 8.2 可维护性
- 组件化结构，视图、逻辑、模拟服务分层清晰。
- 核心业务状态具备明确命名与注释。

### 8.3 兼容性
- 兼容 Chrome/Edge 最新两个大版本。
- 适配 1366x768 及以上主流PC分辨率。

### 8.4 可演示性
- 术语风格统一，日志具备“专业控制台”观感。
- 视频画面稳定播放，支持全屏演示。

## 9. 验收标准（UAT）
- [ ] 打开页面后自动完成一次检测并展示状态。  
- [ ] 点击“一键去控”后弹出居中控制台并开始逐步打印日志。  
- [ ] 日志包含中文步骤说明与英文关键指标（至少3类不同指标）。  
- [ ] 进度条随步骤推进最终达到100%。  
- [ ] 去控完成后出现成功提示。  
- [ ] 监控视频持续播放，且可进入/退出全屏。  
- [ ] 监控视频不受去控流程状态影响。  

## 10. 后续优化建议
- 增加日志级别（INFO/WARN/ERROR）与模块标签（NAS/RRC/DPI/SEC）。
- 增加“导出日志”与“流程回放”功能。
- 增加失败分支与重试机制，提升演示真实性。
